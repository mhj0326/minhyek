1. Project Summary

In this project, the main topic is neural style transfer, a technique that allows you to generate images that combine the content of one image with the style of another. The prokect is divided into three main parts:
1. Content Space Optimization: Starting from random noise, optimize the image to match the content of a given content image. This process involves minimizing the content distance between the generated image and the content image. Its focus is in how to optimize pixels to match a target content.
2. Texture Generation: In this part, it wll focus solely on generating textures. Ignoring the content, optimize the image to match the style of a given style image. This involves understanding how to capture style information using techniques like the Gram matrix, which represents the correlations between different feature maps in a convolutional neural network.
3. Neural Style Transfer: Finally combining the concepts from the previous two parts to perform full neural style transfer. Here, I will optimize an input image to simultaneously match the content of one image and the style of another. This involves defining loss functions that balance content and style distances and then using optimization algorithms like gradient descent to iteratively update the input image until it closely resembles the desired output.

2. Content Reconstruction

The feature set of a given input image, represented as X, at the L-th layer is noted as F^L_X, while for the chosen target content image it is noted as F^L_C. The task's content loss is computed as the aggregate of the squared L2-norm distance among the chosen features of these two images:

Lcontent = Σl ∈ Γ || FlX - FlC ||22,

where \( Γ \) signifies the collective layers within the model designated for content feature extraction. Specifically, for this task, each initial layer within the convolution blocks of VGG19, named \( \texttt{conv1_1} \) through \( \texttt{conv5_1} \), is employed for extracting features.

3. Texture Generation

To determine the style loss, which is a measure of how much the style of our input image differs from that of our target style image, we first rearrange each feature into a two-dimensional format by combining the batch size and channels, and then the height and width. Once we have this new arrangement, we compute what's known as the Gram matrix. This matrix is a special representation that captures the essence of the style of the image by considering how features across the channels relate to each other.

The next step in calculating the style loss involves normalizing this Gram matrix based on the number of elements in each feature, which takes into account the batch size, the number of channels, and the dimensions of the image. After normalization, we compare the Gram matrix of our input image with that of our style target. The style loss is then computed by taking the sum of the differences between the corresponding elements of these two Gram matrices across all layers specified for style extraction. This sum gives us an overall value that represents the disparity in style between the two images.

4. Neural Style Transfer

I choose content layer to be conv_2 and style layer to be conv_1, conv_2, conv_3, conv_4, conv_5. The Gram matrix is meticulously normalized relative to the dimensions of the input, which is a critical step for ensuring that the values within the matrix reflect relative, rather than absolute, correlations between feature activations. This normalization is instrumental in balancing the magnitude of the style loss, particularly when juxtaposed with the content loss. For this specific implementation, the style loss has been weighted significantly higher, by a factor of 100,000, compared to the content loss, which has been assigned a weight of 1. This disparity in weighting underscores the primary focus on style in the optimization process. Over the course of 300 iterations, the LBFGS algorithm—an optimizer known for its efficacy in dealing with large-scale optimization problems—was employed. LBFGS is particularly well-suited for tasks like style transfer due to its capacity for handling non-linear objectives and its requirement for relatively few line searches, which makes it more efficient compared to other gradient descent strategies. The computation time for the optimization process, initiated from a state of random noise versus using the content image as a starting point, was observed to be comparable. This suggests that the initial conditions, whether they be random noise or content-derived, do not significantly affect the computational effort required to achieve convergence, given the architecture and hyperparameters of this model.

Temporal Smoothness for Video

I made create_smooth_transition_gif function to to generate videos from a sequence of images, introducing smooth transitions to enhance visual flow. Utilizing the OpenCV library for image manipulation and the PIL for creating video, this function calculates intermediate frames for each pair of consecutive images. This is achieved by linearly blending the images using OpenCV's addWeighted function, with the blend ratio gradually shifting from the first image to the second. This blending creates a smooth transition effect between images. After adding the original and interpolated frames to a list, the script compiles these into a single video, setting the frame duration and enabling looping. number of frame increased from 54 to 377 through this function
